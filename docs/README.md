## 🔄 反馈文档

**用途**:
- 记录开发过程中遇到的问题
- 记录问题解决方案和经验教训

2. **解决问题后**: 更新反馈文档，记录最终方案


4. **第四步**: 开始工作

   - 遇到问题及时记录反馈
   - 完成后更新进度

## 🤝 协作模式

### Agent Teams 协作

本项目采用 **Claude Code Agent Teams** 模式协作开发：

1. **Team Leader**: 负责整体协调和进度跟踪
2. **Development Agents**: 负责具体功能开发
3. **Review Agents**: 负责代码审查和质量保证
4. **Documentation Agents**: 负责文档更新和维护

### 协作流程

```
┌─────────────────────────────────────────────────────────┐
│  1. Team Leader 分配任务                                 │
│     ↓ 查看设计文档和反馈文档                             │
│     ↓ 分配 Phase 和具体任务                              │
└─────────────────────────────────────────────────────────┘
                         ↓
┌─────────────────────────────────────────────────────────┐
│  2. Development Agents 开发                              │
│     ↓ 阅读设计文档相关章节                               │
│     ↓ 检查反馈文档中的已知问题                           │
│     ↓ 实施功能开发                                       │
│     ↓ 遇到问题记录反馈                                   │
└─────────────────────────────────────────────────────────┘
                         ↓
┌─────────────────────────────────────────────────────────┐
│  3. Review Agents 审查                                   │
│     ↓ 检查代码质量                                       │
│     ↓ 验证是否符合设计文档                               │
│     ↓ 提出改进建议                                       │
└─────────────────────────────────────────────────────────┘
                         ↓
┌─────────────────────────────────────────────────────────┐
│  4. Documentation Agents 更新文档                        │
│     ↓ 更新反馈文档                                       │
│     ↓ 更新进度跟踪                                       │
│     ↓ 记录经验教训                                       │
└─────────────────────────────────────────────────────────┘
```
